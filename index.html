Fast inference ≠ kernels

Once the matmul is fast,
everything else becomes the bottleneck.

CPU scheduling. Memory layout. Tokenization. Image preprocessing.

War stories

  • Pillow's resize was eating every request — wrote our own SIMD image resizer.
  • F.linear had wrapper overhead. Bypassed PyTorch, called cublasLt directly.
  • Custom MoE kernel beats Triton ~25% at 4k tokens (CuTe / WGMMA).
  • Cold start: 14.4s → 2.7s. 5.4× — matters for serverless.
  • cudaGetDeviceProperties is slow enough to cache. Even OS calls add up.
  • Wrote our own kernel dispatch (pybridge → DLPack) to skip the PyTorch dispatcher.
  • CUDA graphs for the vision encoder. Capture once, replay forever.
  • KV cache eviction: O(n) → O(1) by tracking evictable counts incrementally.
